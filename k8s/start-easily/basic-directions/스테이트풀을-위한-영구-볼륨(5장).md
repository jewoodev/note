애플리케이션에는 고객 정보나 주문 정보처럼 삭제되지 않아야 하는 정보들도 있다. 그러한 데이터들을 보관하는 저장소를 볼륨이라고 하며, 데이터를 저장소에 저장해 두는 애플리케이션을 스테이트풀이라고 한다. 저장소를 사용하지 않는 스테이트리스는 단순히 디플로이먼트를 배포하면 되기 때문에 구성이 간단하지만, 스테이트풀은 저장소를 사용해야 하기 때문에 구현이 복잡하다. 

# 볼륨
볼륨은 데이터를 저장하는 저장소를 말한다. 그리고 이게 파드(컨테이너)가 종료되어도 데이터를 보존할 수 있는 방법이 된다. 볼륨은 데이터가 저장되는 위치에 따라 세 가지로 분류할 수 있다. 

- **파드 내에 위치**: 파드 내에 데이터를 저장한다면 파드가 종료될 때 데이터도 함께 삭제된다.
- **워커 노드 내에 위치**: 워커 노드 내에 데이터가 저장된다면 파드가 종료되어도 데이터가 유지되지만, 노드가 종료되면 데이터도 삭제된다.
- **노드 외부에 위치**: 파드와 노드가 종료되어도 데이터는 항상 보존된다.

## 볼륨 유형
볼륨은 파드의 구성 요소이므로 컨테이너와 마찬가지로 매니페스트로 정의해 사용한다. 하지만 볼륨은 독립적인 쿠버네티스 리소스가 아니기 때문에 자체적으로 생성되거나 삭제될 수 없다. 볼륨은 파드뿐만 아니라 파드 내에 모든 컨테이너에서 사용할 수 있으며 접근하려는 각 컨테이너에 마운트해야 한다.

볼륨은 임시 볼륨, 로컬 볼륨, 외장 볼륨 세 가지 유형으로 나뉜다. 각 유형을 살펴보면 다음과 같다.

- 임시 볼륨(emptyDir): 파드 내의 공간을 사용하는 것으로, 파드가 삭제(종료)되는 즉시 함께 삭제됨
- 로컬 볼륨(hostPath): 노드 내의 디스크를 저장소로 사용하는 것으로, 노드가 삭제되는 즉시 함께 삭제됨
- 외부 볼륨: 노드 외부에 있는 외부 저장소를 이용하는 것으로 파드나 노드와 무관하게 영구적으로 유지되나, 별도의 유지에 비용이 듬

# 영구 볼륨과 영구 볼륨 클레임
개발자가 디플로이먼트로 파드를 생성할 때 볼륨을 정의한다면, 볼륨 자체를 정의하는 것이 아니라 볼륨을 요청하는 볼륨 클레임을 지정하는 것이다. 그리고 그 볼륨 클레임에서 볼륨을 할당한다. 영구 볼륨을 요청하는 볼륨 클레임을 영구 볼륨 클레임(Persistent Volume Claim, PVC)이라고 한다.

더 자세히 말하면 개발자가 요청한 영구 볼륨(영구 볼륨 클레임)은 API 서버로 보내지고 이후 API 서버를 통해 영구 볼륨이 할당된다. 

개발자가 파드를 생성할 땐 볼륨을 요청하는 영구 볼륨 클레임을 정의하면 볼륨이 자동으로 할당된다. 실제로 마운트(연결)되는 볼륨이 무엇인지 알 필요 없이 영구 볼륨 클레임을 명시하기만 하면 되기 때문에 개발자는 볼륨 자체에 대해서는 신경쓰지 않고 볼륨을 사용할 수 있다. 이를 통해 저장소의 볼륨은 시스템 관리자만 관리하면 되는 구조를 만들 수 있어 역할 분리가 쉬워진다.

## 영구 볼륨과 영구 볼륨 클레임 사용하기
파드에서 영구 볼륨 클레임의 이름을 `persistenceVolumeClaim` 을 이용해 지정하고, 해당 이름을 갖는 영구 볼륨 클레임에서 `storageClassName` 을 이용해 영구 볼륨에 바인딩(볼륨을 할당)할 수 있다. 

이번 예시에서 mysql 데이터베이스 파드에 영구 볼륨을 마운트한다. 외부 저장소를 사용하는 방법도 이와 유사하기 때문에 이를 참고하자.

```
# 파드(디플로이먼트)
volumeMounts: /var/lib/mysql

persistenceVolumeClaim:
  claimName:mysql-pv-claim
  
# 영구 볼륨 클레임(PVC)
metadata:
  name: mysql-pv-claim

- resources:
    requests:
      storage: 20Gi
- storageClassName: manual

# 영구 볼륨(PV)
storageClassName: manual

- capacity:
-   storage: 20Gi
- hostPath:
-   path: "/volume/data"    
```

먼저 영구 볼륨을 위한 `pv.yaml` 파일을 생성한다.

```yaml
apiVersion:
kind: PersistentVolume
metadata:
  name: mysql-pv-volume
  labels:
    type: local
spec:
  storageClassName: manual # 영구 볼륨 클레임의 요청을 해당 영구 볼륨에 바인딩
  capacity:
    storage: 20Gi # 영구 볼륨으로 20Gi 할당 
  accessModes:
    - ReadWriteOnce
  hostPath:
    path: "/mnt/data" # 외부 저장소가 있다면 대체할 것. 노드의 /mnt/data 디렉터리를 볼륨으로 사용
```

```bash
kubectl apply -f pv.yaml
```

이번에는 영구 볼륨 클레임을 생성한다.
