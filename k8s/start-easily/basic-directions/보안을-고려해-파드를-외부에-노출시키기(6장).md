# 파드를 외부에 노출시키기
파드는 클러스터 안에서 노드를 옮겨다니는 특징이 있다. 이런 특징은 파드의 IP를 동적으로 계속해서 변화시키기에 외부에서 접근하기가 어렵다. 쿠버네티스는 쿠버네티스의 서비스로 그러한 어려움을 해소해준다. 서비스를 사용하면 파드가 클러스터 내 어디에 있던지 고정된 IP로 접근하는 것처럼 쉬운 접근이 가능하다.

## ClusterIP
ClusterIP는 기본 서비스 타입으로, 클러스터 내 파드에 접근할 수 있는 가상의 IP이다. 서비스는 기본적으로 이걸 갖는다. 그리고 서비스를 파드에 연결해놓으면 서비스 IP(ClusterIP)를 통해 파드에 접근할 수 있다. 이것은 클러스터 내부에서만 사용할 수 있으며 외부에서는 사용할 수 없다.

ClusterIP의 서비스를 정의하는 매니페스트를 작성할 땐 서비스 유형을 지정하지 않는다. 지정하지 않으면 ClusterIP로 사용하겠다는 의미가 되기 때문이다.

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: clusterip-nginx
spec:
  selector:
    matchLabels:
      run: clusterip-nginx
  replicas: 2
  template:
    metadata:
      labels:
        run: clusterip-nginx
    spec:
      containers:
      - name: clusterip-nginx
        image: nginx:latest
        ports:
        - containerPort: 80
```

```bash
kubectl apply -f ClusterIP.yaml
```
이렇게 어플라이 명령어를 이용해 디플로이먼트(파드)를 생성한다. 생성된 파드는 각각 고유 IP를 갖는다. 이제 서비스를 생성하자.

```bash
kubectl expose deployment/clusterip-nginx
```

## ExternalName
ExternalName은 조금 특별한 경우에 사용하는 서비스이다. 노드포트, 로드밸런서 등이 외부에서 클러스터 내부의 파드에 접속하기 위한 서비스라면, ExternalName은 클러스터 내부에서 외부의 엔드포인트에 접속하기 위한 서비스이다. 여기서 외부의 엔드포인트는 데이터베이스나 애플리케이션의 API 등을 의미한다.

이는 다음과 같이 사용한다.

```yaml
kind: Service
apiVersion: v1
metadata:
  name: external-service
spec:
  type: ExternalName
  externalName: myservice.test.com # external-service와 연결하려는 외부 도메인 값을 설정
```

앞에서 정의한 external-service와 myservice.test.com은 다음과 같은 의미를 갖는다.

- **외부 FQDN (Fully Qualified Domain Name)**: myservice.test.com
- **CNAME**: external-service

즉, external-service라는 이름으로 요청이 들어오면 클러스터 외부에 존재하는 myservice.test.com이라는 주소를 반환하는 것이 ExternalName 서비스다. 이런 ExternalName 서비스는 프록시나 특정 이름(혹은 주소)을 다른 이름으로 자동 변환하는 리다이렉트 용으로 사용한다.

## 노드포트
모든 워커 노드에 특정 포트(노드포트)를 열고 그곳으로 들어오는 모든 요청을 노드포트 서비스로 전달한다. 그 후 노드포트 서비스는 해당 업무를 처리할 수 있는 파드로 요청을 전달한다. 즉, 노드포트 타입의 서비스는 외부 사용자(혹은 애플리케이션)가 클러스터 내부에 있는 파드에 접속할 때 사용하는 서비스 중 하나다. 

노드포트 서비스를 한 번 만들어보자. 그러려면 파드를 먼저 만들어야 하는데, 다음과 같이 매니페스트를 작성하자.

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  labels:
    app: nginx
spec:
  replicas: 2
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
        - containerPort: 80
```

이걸 기반으로 디플로이먼트를 생성하고, 다음의 서비스 매니페스트로 서비스를 생성하자.

```yaml
apiVersion: v1
kind: Service
metadata:
  name: my-nginx
  labels:
    run: my-nginx
spec:
  type: NodePort # 외부에서 접속할 때 노드포트 서비스 사용
  ports:
  - port: 8080 # 노드포트 서비스에서 사용하는 포트
    targetPort: 80 # 파드에서 사용하는 포트
    protocol: TCP
    name: http
  selector:
    app: nginx
```

위의 매니페스트에서는 외부에서 접속할 때 사용할 서비스의 노드포트는 별도로 지정하지 않아 임의로 부여된다.
