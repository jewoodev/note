컴퓨터의 디렉토리의 관계처럼 1:M 관계를 갖는 테이블 A(1)와 B(M)가 있을 때 B와 1:M 관계를 갖는 테이블이 있고, 또 그 테이블에도 1:M 관계를 갖는 테이블이 있는 경우에 "1:M 재귀적 관계가 있다." 라고 표현할 수 있다.

이 경우는 재귀적으로 무한하게 관계가 가능한 경우이기 때문에 1:M 관계가 생겨날 때마다, 혹은 이미 많이 있는 1:M 관계를 설계하고 관리하기가 어려워진다. 이럴 땐 자기 참조 외래 키를 갖는 테이블로 설계해 이를 용이하게 관리할 수 있다.

```sql
-- MySQL 9.5.0
CREATE TABLE department (
    department_id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    department_name VARCHAR(50),
    high_department_id INT UNSIGNED,
    CONSTRAINT fk_high_department_id FOREIGN KEY (high_department_id) REFERENCES department(department_id)
);
```
