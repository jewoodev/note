# 1:M 관계
- 외부의 다른 테이블의 primary key는 foriegn key가 될 수 있다.
  - 외부의 다른 테이블의 primary key를 foriegn key로 갖는 테이블은 **자식 테이블**
    - primary key를 foriegn key로 제공한 테이블은 **부모 테이블**
- 부모 없는 자식은 존재하지 않듯이, 부모 없는 자식은 발생하지 않도록 RDBMS가 막아준다.
  - 부모 테이블에 없는 primary key를 foriegn key로 갖는 자식 테이블은 없다. 
- 이는 마스터 테이블 두 개의 관계이다.
  - 즉, 명사로써 존재하는 테이블 두 개의 관계이다.
  - 이 관계를 문장으로 표현할 땐 "has", 갖는 소유하는 표현이 들어간다.

# 식별 / 비식별 관계
- 식별 관계 (Identifying Relationship)
  - 부모 테이블의 기본 키가 자식 테이블의 기본 키의 일부가 되는 관계
  - 자식이 부모 없이 존재할 수 없음 (강한 종속성)
  - 예: `ORDER_ITEM(order_id, item_seq)` - order_id가 PK의 일부
- 비식별 관계 (Non-Identifying Relationship)
  - 부모 테이블의 기본 키가 자식 테이블의 일반 외래 키로만 존재
  - 자식이 독립적인 기본 키를 가짐
  - 예: `ORDER_ITEM(item_id, order_id)` - order_id는 FK일 뿐
- **관계 모델링 관점**에서의 표현

# 독립형 / 상속형 기본 키
- 독립형 기본 키 (Independent Primary Key)
  - 자식 테이블이 부모와 무관한 **독립적인 기본 키**를 가짐
  - 외래 키는 일반 컬럼으로만 존재
  - 예: `ORDER_ITEM(item_id PK, order_id FK, ...)`
- 상속형 기본 키 (Inherited Primary Key)
  - 부모의 기본 키를 **상속받아** 자식의 기본 키에 포함
  - 복합 키 구조가 됨
  - 예: `ORDER_ITEM(order_id PK/FK, item_seq PK, ...)`
- **기본 키 설계 관점**에서의 표현
