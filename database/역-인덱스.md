# Inverted Index
전통적인 RDBMS에서는 TEXT 컬럼에 특정 문자열이 포함된 레코드를 조회할 때 **like** 검색을 하기 때문에 데이터가 늘어날수록 검색해야 할 대상이 늘어나 시간이 오래 걸리고, row 안의 내용을 모두 읽어야 하기 때문에 속도도 느리다. 

역 인덱스는 **책 맨 뒤에 있는** 주요 키워드에 대한 내용이 몇 페이지에 있는지 볼 수 있는 **찾아보기 페이지**와 같다. 이 기능을 제공하는 ElasticSearch는 각 추출된 키워드를 **텀(term)** 이라고 부른다. 역 인덱스가 있으면 특정 키워드를 포함하고 있는 도큐먼트들의 **id**를 바로 얻어올 수 있다.
