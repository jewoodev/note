데이터베이스에서 뷰(View)는 하나 이상의 테이블에서 SELECT 쿼리를 미리 정의해두고, 그 결과를 마치 하나의 테이블처럼 사용할 수 있게 만든 가상 테이블

# Example of a view
```sql
CREATE VIEW active_users AS
SELECT id, name, email
FROM users
WHERE status = 'ACTIVE';
```

이렇게 만들어두면 이후에 SELECT * FROM active_users처럼 일반 테이블 쓰듯이 조회할 수 있다.

중요한 점은 뷰 자체가 데이터를 저장하는 건 아니라는 것! 조회할 때마다 내부적으로 원래 정의된 쿼리가 실행된다. 그래서 원본 테이블의 데이터가 바뀌면 뷰의 결과도 자동으로 바뀐다.

# 어떨 때 사용하는게 좋을까?
- 복잡한 조인이나 서브쿼리를 자주 반복해야 할 때
  - 쿼리를 뷰로 만들어두면 재사용이 편함
- 특정 컬럼만 노출하고 싶을 때
  - 민감한 정보가 있는 컬럼을 제외한 뷰를 만들어서 접근 제어 용도로 쓸 수 있음
- 비즈니스 로직을 DB 레벨에서 캡슐화하고 싶을 때
  - 예를 들어 "활성 사용자"의 정의가 바뀌면 뷰 하나만 수정하면 됨

참고로 MySQL에서는 단순한 뷰에 대해 INSERT/UPDATE도 가능하긴 한데, 조인이나 집계가 포함된 뷰는 읽기 전용이다. 실무에서는 대부분 조회 용도로 쓴다고 보면 된다.
