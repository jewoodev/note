# 레플리케이션
스케일-아웃을 하는데 있어 가장 대표적으로 사용되는 기술이다. 이는 쉽게 말하면 데이터베이스를 여러 개 복사해서 거의 동일한 데이터를 가진 복제본을 만드는 기술이다.

복제가 이루어지면 MySQL 서버는 마스터 노드와 레플리카 노드로 구성되며, 마스터는 메인 노드로써 데이터의 생성 & 수정 & 삭제같은 모든 쓰기 작업이 이루어지는 주체이다.  
레플리카는 마스터에 추가된 데이터를 주기적으로 복사하면서 데이터의 일관성을 보장한다.  
마스터는 쓰기 작업에만 집중하고 읽기 작업은 대부분 여러 개의 레플리카가 하게 해서 처리량을 높인다.

MySQL 서버에서 발생하는 모든 변경 사항은 별도의 로그 파일에 순서대로 기록된다. 이를 바이너리 로그라고 한다.  
바이너리 로그에 기록된 각 변경 정보들을 이벤트라고도 한다.  
마스터 노드에서 생성된 바이너리 로그가 레플리카 서버로 전송되고, 레플리카 서버에서는 받은 로그를 로컬 디스크에 저장한 뒤 자신이 가진 데이터에 반영함으로써 데이터 동기화가 이루어진다.

마스터 노드에 장애가 발생하면 슬레이브 노드 중 하나가 마스터 노드가 되어 가용성을 유지한다.
