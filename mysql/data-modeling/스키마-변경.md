# Online DDL
많은 레코드가 축적된 테이블의 스키마를 변경하기 시키는 ALTER TABLE 쿼리는 수십 분의 다운 타임과 큰 크기의 디스크 공간을 필요로 한다.  
ALTER TABLE은 테이블을 복사하는 방식으로 처리된다. 이 처리 방식의 비효율을 Online DDL을 이용해 완화할 수 있는데 모든 것들에 대해서 가능한 것은 아니다.  
그래서 스키마 변경없이 확장성을 가져갈 수 있는 다른 접근법, Json 활용이나 EAV 전략을 사용하는 것이다. 

ALTER TABLE은 테이블 전체에 기본적으로 X Lock을 건다. 모든 작업을 차단시키고 새 스키마를 생성한 후 모든 데이터를 새 테이블에 삽입하고 인덱스를 재생성하는 것이다. 하지만 별도로 옵션을 주어 락과 알고리즘을 선택할 수 있다.

```sql
ALTER TABLE <테이블명> ADD COLUMN <컬럼명> <옵션>, ALGORITHM=INPLACE, LOCK=NONE;
-- MySQL 5.6 이상에서 지원, Online DDL임
```
이 온라인 DDL을 사용해 다운타임을 최소화하고 동시에 읽기와 쓰기를 가능하게할 수 있지만 여전히 성능저하는 발생할 수 있다. 

- 알고리즘 인수값
  - COPY: 기존 방식으로 테이블을 복사
  - INPLACE: 인덱스를 재구성하는 방식 (Online DDL)
  - DEFAULT: MySQL이 자동으로 선택
- 락 인수값
  - NONE: 락 없이 실행
  - SHARED: 공유락 사용
  - EXCLUSIVE: 베타락 사용
  - DEFAULT: MySQL이 자동으로 선택

## Online DDL이 지원하지 않는 것
- 데이터 타입 변경
- 컬럼 이름 변경
- 기본값 변경

## Online DDL이 지원하는 것
- 인덱스 추가 & 삭제
- 컬럼 추가 & 삭제
