`EXPLAIN`으로 확인 가능한 실행 계획에서 파악할 수 있는 정보들은 다음과 같다.

# `type`
`type` 컬럼은 해당 쿼리가 데이터를 어떤 방식으로 스캔하였는지에 대한 정보이다. const, eq_ref가 가장 빠르며, ALL(Full Table Scan)은 가장 느리다. ref, range, index 순으로 성능이 저하된다.

## 주요 type 종류 (성능 높은 순)
- const:
  - 프라이머리 키(PK)나 유니크 키(Unique Key)를 사용하여 단 1건의 행만 매칭되는 경우. 
    - 성능이 가장 빠름.
- `eq_ref`:
    - 조인할 때 프라이머리 키나 유니크 키를 사용하여 딱 1건의 데이터만 조인되는 경우.
- `ref`:
  - 인덱스를 사용하여 1건 이상의 데이터가 매칭될 때(Unique가 아닌 인덱스). 
    - 상당히 성능이 좋음.
- `range`:
  - 인덱스를 사용하여 지정된 범위의 데이터를 검색할 때 (<, >, IN, BETWEEN 등).
- `index`:
  - index_merge처럼 인덱스를 사용하지만 테이블 전체를 인덱스 트리 순서대로 스캔할 때.
    - ALL보다 낫지만 여전히 느림.
- `ALL`:
  - Full Table Scan. 인덱스를 사용하지 않고 처음부터 끝까지 전체 테이블을 읽음. 
    - 성능이 가장 낮으며 튜닝 대상.
