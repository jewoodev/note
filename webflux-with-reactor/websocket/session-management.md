웹소켓 연결을 관리할 때 핵심적인 **세션 관리**는 어떻게 하면 좋을지 알아보자. 

일단, 세션이라는 데이터는 휘발적이므로 이에 어울리며 성능 이점을 가져갈 수 있는 메모리 기반 스토리지를 사용하는 것이 적합하다. 그러한 것으로 대표적인 Redis가 있겠다.

만약 메신저 서버에서 채팅방에 대해 세션을 관리하는 전략을 세워 사용한다면 (Redis가) 더욱 적합하다. Redis Pub/Sub을 이용해 채팅방으로 생성되는 Set 데이터(채팅방을 식별할 Key값을 갖는)를 구독하고, 웹소켓 연결 수립이 이루어질 때마다 '방에 입장한 유저 정보'를 Set 데이터로 저장하고, 채팅 메세지가 웹소켓 연결을 통해 전달될 때마다 그 메세지를 발행하면 되기 때문이다.
