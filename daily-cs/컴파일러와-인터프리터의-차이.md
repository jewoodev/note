컴파일러와 인터프리터는 둘 다 개발자가 작성한 프로그래밍 언어(C++, Java, Python 등)를 컴퓨터가 이해할 수 있는 기계어로 번역해주는 '번역기'이다. 

둘의 가장 큰 차이는 '번역하는 방식'과 '시점'에 있다.   
컴파일러는 우리가 작성한 코드 전체를 한꺼번에 읽어서, 통째로 기계어로 번역한 뒤 실행 파일을 만들어낸다. 번역 과정(컴파일)은 오래 걸릴 수 있지만, 일단 번역된 실행 파일이 만들어지면 그 다음부터는 매우 빠르게 실행된다. C언어, C++, Java 등이 대표적이다.   
반면 인터프리터는 코드 전체를 한 번에 번역하지 않고, 한 줄씩 읽으면서 바로바로 번역하고 실행한다. 전체 코드를 번역하는 시간이 필요 없어서 실행 시작이 빠르고 수정하기도 편하지만, 매번 한 줄씩 번역해야 하니 실행 속도는 컴파일러 방식보다 느릴 수 있다. Python, JavaScript 등이 여기에 속합니다. 

둘의 차이점을 책 번역 과정에 비유할 수 있다. 컴파일러는 책 한 권(코드 전체)을 완벽하게 번역해서 '한국어판 책(실행 파일)'을 출간하는 방식이다. 번역은 오래 걸렸지만, 일단 출간되면 독자들은 빠르게 읽을 수 있다. 인터프리터는 책을 읽어달라는 요청이 올 때마다 '한 문장씩 읽고 바로 통역'해주는 방식이다. 바로바로 들을 순 있지만, 매번 통역을 거쳐야 하니 책을 다 읽는 데는 시간이 더 걸린다.
