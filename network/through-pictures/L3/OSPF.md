# OSPF
> Open Shortest Path First

AS 내에서 사용되는 Link State 방식의 동적 라우팅 프로토콜이다. 각각의 라우터들은 자신과 연결된 모든 이웃 라우터들에게 LSA를 전달한다. LSA는 Link State Advertisement의 줄임말로 자신의 링크 정보를 광고하는 메세지이다. LSA에는 광고하는 라우터의 ID와 포트의 IP 주소, 비용 등이 저장되어 있다. 여기서 비용이란 연결된 케이블의 속도를 말한다(속도가 빠를수록 비용이 적다). 

하나의 라우터가 LSA 전송을 하면 수신한 라우터들은 자신의 인접한 라우터들에게 전송한다. 이 때 중복으로 LSA를 받은 라우터는 늦게 도착한 LSA를 파기한다. 그런 방식으로 모든 라우터는 동시다발적으로 자신의 LSA를 연결된 모든 라우터들에게 전달한다. 전달된 LSA는 각 라우터의 Link State Data Base(LSDB)에 저장된다. 

모든 라우터가 LSA를 교환하고 나면 모두 같은 LSDB를 갖게 된다. 그러면 그 LSDB를 이용해서 최단 경로를 계산한다. 최단 경로는 다익스트라 알고리즘을 이용해 계산한다. OSPF는 LSA를 모든 라우터들에게 전달하고 다익스트라 알고리즘을 이용해 각자 라우팅 테이블을 만든다.

## 장점
- 한 라우터에 변화가 생겼을 때 다른 모든 라우터들에 업데이트되는 시간이 짧다.
  - 커버리지 타임이 짧다.
- 이웃 라우터와 자주 통신하는 RIP와는 다르게 링크 정보를 자주 교환하지 않아서 트래픽을 줄일 수 있다.
- 홉의 제한이 없어 대규모 네트워크에서 사용 가능하다.

## 단점
- 비용이 비싸다.
  - 라우터가 모든 라우팅 정보를 저장하기 때문에 많은 메모리가 필요하다.
  - 최단 거리를 계산하기 위해 CPU가 더 바빠진다.
- 다른 라우팅 프로토콜보다 복잡해 러닝커브가 있다.
