# ICMP
- Internet Control Message Protocol, 줄임말
- 네트워크에서 해당 목적지까지 데이터가 도달할 수 있는지,
- 도달할 수 없다면 그 원인이 무엇인지 알아내기 위한 프로토콜
- ICMP 데이터가 전송될 때
  - ICMP 헤더, IP 헤더, 이더넷 헤더가 붙어서 전송됨
    - ICMP 헤더엔
      - 현재 ICMP가 어떤 타입의 ICMP인지 나타내는 8비트로 이루어진 타입필드가 있음
      - 만약 목적지까지 도달할 수 없다면 3이라는 숫자가 설정됨
- ping 명령어가 ICMP 메세지를 보내는 명령어이다.

## ICMP 헤더의 타입필드
타입의 유형은 여러가지가 있는데 몇가지만 살펴보자.

- 0
  - 에코 응답을 보낼 때 사용한다.
  - 만약 우리가 목적지에 ping 요청을 했다면 응답 메세지는 Type 0번으로 설정되어 돌아온다. 
- 3
  - 목적지에 도달이 불가할 때 사용한다.
- 8
  - 우리가 에코 요청을 보낼 때 사용한다.
- 11
  - 요청한 패킷이 목적지에 도달하는데까지 시간이 초과되었을 때 사용한다.

## 코드필드
타입필드 다음으로 8비트로 이루어진 필드. 타입필드에 대한 자세한 설명을 하는 필드이다. 만약 타입필드가 3번이고 코드필드가 0번이면 코드 0번으로 네트워크에 도달할 수 없다는 걸 알 수 있다. 코드의 유형도 여러가지가 있다.

## 체크섬, 나머지 헤더
- 체크섬
  - 16비트로 이루어짐
  - 메세지에 오류가 있는지 체크하는 필드
- 나머지 헤더
  - 타입과 코드에 따라 변경되는 필드이다.
