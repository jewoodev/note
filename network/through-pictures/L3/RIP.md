# RIP
> RIP(Routing Information Protocol)

AS 내에서 사용되는 Distance Vector 방식의 동적 라우팅 프로토콜이다. RIP는 '목적지 라우터까지의 거리'와 "목적지까지 가기 위해서 어떤 인접 라우터 방향으로 가는지"를 라우팅 테이블에 저장한다. 그리고 저장된 정보를 인접 라우터와 30초마다 공유하는 방식으로 라우팅 테이블을 유지한다. 

## 동작 흐름
1. 처음에 각 라우터의 라우팅 테이블은 포트에 연결이 되어있는 인터페이스 정보와 홉 정보를 가지고 있다.
2. 인접한 라우터와 정보를 공유하여 IP 주소가 같으면 필터링하고 저장한다. 
   - 저장할 때 인접한 라우터가 어떤 포트로 연결되어 있는지 확인해 같은 포트 값으로 저장한다.
   - 인접한 라우터의 정보를 가져올 때 홉 값을 1 올려서 저장한다.
   - 한 번 공유한 시점에서는 인접한 라우터들끼리만 데이터가 공유된 상태이다.
     - 하지만 다음 공유가 진행되면 두 홉 떨어진 라우터들끼리도 공유가 된다.
3. 다시 공유를 진행한다.
   - 라우팅 테이블에 변화가 있는 라우터만 인접한 라우터에 공유가 된다.
   - 저장할 때 인접한 라우터가 어떤 포트로 연결되어 있는지 확인해 같은 포트 값으로 저장한다.
   - 인접한 라우터의 정보를 가져올 때 홉 값을 1 올려서 저장한다.

## 커버리지 타임
라우팅 테이블에 변화가 생겼을 때 모든 라우터에 업데이트가 완료되는 시간을 말한다.

## 장점
- 하나의 라우터가 모든 정보를 가지고 있지 않기 때문에 적은 메모리를 요구한다.
- **구성이 간단하다**.
- 표준으로 사용하는 곳이 많다.
- 대부분의 라우터에서 지원한다.

## 단점
- 라우팅 테이블에 변화가 없어도 30초 마다 인접 라우터와 라우팅 테이블을 교환하기 때문에 쓸데없는 트래픽이 발생한다.
- 업데이트된 라우팅 테이블의 정보가 하나의 홉을 건너기 위해서는 30초라는 시간이 걸립니다.
- 경로를 나타내는 정보가 다음 라우터와 홉밖에 없는 디스턴스 벡터 방식이므로
  - 전체 경로는 알 수 없다.
    - 따라서 순환경로에 빠지는 문제가 발생할 수 있다.
    - 이런 이유로 대규모 네트워크에서는 사용하기 어렵고
      - 소규모 네트워크에서 주로 사용되는 라우팅 프로토콜이다.
