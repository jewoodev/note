# 스위치란?
- 스위치는 브릿지와 기능적 차이는 없고, 단지 성능이 더 좋아진 것
- '스위칭 허브'라고도 불림
  - 브릿지 = 스위치 = 스위칭 허브
- MAC 주소를 이용하기 때문에 데이터 링크 계층에 속함

# 필터링
- 만약 스위치가 수신한 이더넷 프레임의 '출발지와 목적지 MAC 주소의 포트 번호'가 같으면
    - 출발지의 MAC 주소와 포트 번호를 Learning 한 후
    - 같은 포트 번호 임을 인지하면
    - 그 두 호스트는 같은 콜리전 도메인에 있는 것이므로 그 콜리전 도메인 안에서 통신되었음을 알아차림
    - 어디에도 데이터를 전송하지 않음
        - 이걸 필터링이라고 함

# 에이징
- 시간이 지나 많은 기기의 정보가 저장되었다면 메모리가 부족해질 것임
  - 따라서 일정 시간이 지나면 메모리에서 자동으로 제거함
  - 이를 **Aging**이라고 부름
  - 보통 5분이 지나면 삭제
- Aging Timer가 다 되기 전에 MAC 주소를 참조한다면 
  - 해당 MAC 주소의 Aging Timer는 초기화
    - 즉, 시간이 연장됨

# 브릿지와의 (구체적인) 차이점
- 더 많은 포트를 가지고 있음
- ASIC이라는 하드웨어로 스위칭 작업을 수행
  - 소프트웨어적으로 처리하는 브릿지보다 속도가 더 빠름
- 오류를 체크하는 기능이 있음
  - 브릿지는 x
- 버퍼를 가지고 있음
  - 브릿지는 x

# 최근엔
- 스위치 가격이 싸져서 브릿지를 쓰는 일은 거의 없음
- 브릿지와의 경계가 사라져 스위치라는 단어을 더 많이 사용
