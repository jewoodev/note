# 스위치로 만든 LAN의 문제 케이스
4개의 PC가 있고 사각형 토폴로지 구조로 각 PC에 스위치가 하나씩 연결되어 스위치끼리 각각 두 개의 링크를 가진 LAN이 있다고 하자.

하나의 PC에서 데이터를 Flooding하면 결과적으로 대각선 위치의 PC가 두 개의 프레임을 받게되고 그걸 다시 플러딩하고, 반대쪽에서도 같은 현상이 나타나 다시 플러딩해서 같은 플러딩이 또 일어나면서 점점 네트워크에서 프레임들이 쌓이다가 결국 네트워크가 다운되버리는 상황까지 오게 된다.

## 문제 해결법
- 연결된 4대의 스위치에 사이클이 생기지 않도록 한 쪽 연결을 끊어주는 것
- 사람 손으로 일일이 하기엔 번거로움
  - 스위치가 대신 이 작업을 수행
    - '**스패닝 트리 프로토콜**'이란 걸 이용해서 

# 스패닝 트리란?
트리 자료 구조에서 "모든 노드를 사이클 없이 연결하는" 트리를 말한다.

# 스패닝 트리 프로토콜이란?
1. 여러 스패닝 트리를 구하고
2. 구한 스패닝 트리 중 '가장 빨리 도달할 수 있는 스패닝 트리(**최소 스패닝 트리**)'로 데이터를 전송해
3. 무한 루프에 빠지지 않게 하는 프로토콜


- 최소 스패닝 트리를 구하는 방법은 여러 알고리즘이 있음
  - 그 중 하나가 최소 신장 트리
