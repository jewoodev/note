- UDP를 사용하면 메세지에 UDP 헤더가 붙음 
- UDP는 신뢰성은 없지만 자유로운 프로토콜
  - 재전송을 하지 않고, 순서가 뒤바뀐 데이터를 재조립하지 않음
  - 커넥션 확립 과정이 없음
  - 일방적으로 데이터를 전송함
  - 데이터가 분실되는 경우 프로그래머가 직접 애플리케이션 계층에서 요청해야 함
    - 신뢰성을 보장하려면 프로그래머의 실력이 중요함
- 신뢰성을 보장하는 여러 절차를 생략하기에 
  - 고속으로 처리하는 용도를 가짐

# UDP 헤더
- Source Port(16 bits), Destination Port(16 bits) 
- UDP Length(16 bits)
  - UDP 헤더와 데이터의 크기를 합친 값이 저장됨
- Checksum(16 bits)
  - UDP 헤더와 데이터가 변조됐는지 체크하기 위한 용도
