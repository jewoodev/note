# 트랜스포트 계층이란?
- 호스트로 도착한 데이터를 목적지 애플리케이션으로 정확히 전달하는 역할을 하는 계층
- 호스트 내의 애플리케이션을 구분하는 용도로 포트(Port)라는 것을 사용함
  - 데이터링크 계층에선 MAC 주소
  - 네트워크 계층에선 IP 주소로 호스트를 구분하였음
  - 포트는 $2^{16}$으로, 부호없는 정수 0부터 65535번이 될 수 있음
  - 랜카드나 라우터에 있는 포트와는 다른, 논리적인 개념임
  - 쉽게 표현하면 '프로세스의 주소'
  - 포트번호
    - 0 ~ 1023: 잘 알려진 포트(Well-known Port)
      - IANA에서 미리 할당한 포트, 특정한 서비스에 사용됨
        - 80번: HTTP (웹 서버)
        - 443번: HTTPS (보안 웹 통신)
        - 25번: SMTP (이메일 발신)
        - 23번: 텔넷
    - 1024 ~ 49151: 등록된 포트(Registered Port)
      - 특정 애플리케이션 서비스에 사용하도록 등록할 수 있는 포트
    - 49152 ~ 65535: 동적 포트(Dynamic Port)
      - 다른 사람이 임의로 사용하는 것을 피하면서 임시로 할당되는 포트
