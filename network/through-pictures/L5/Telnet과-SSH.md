두 기술은 원격 로그인을 구현하는데 쓰이는 프로토콜이다. 두 기술을 이용하면 다른 컴퓨터 앞에 직접 가지 않아도 네트워크로 연결되어 멀리 떨어진 컴퓨터에도 로그인할 수 있다.

# Telnet
Telnet은 TCP 커넥션 하나를 이용해 원격 컴퓨터에 로그인하고 (직접 쓰고 있는) 컴퓨터 A에서의 입력이 (멀리 떨어진) 컴퓨터 B로 전송되어 실행되는 것이다. Telnet 덕분에 네트워크 관리자들은 물리적으로 떨어져 있는 라우터나 스위치같은 장치에 직접 가지 않고 사무실에서 로그인해 네트워크 설정을 할 수 있게 되었다. 이뿐만 아니라 우리가 사용하고 있는 공유기도 Telnet을 이용해 로그인할 수 있는데 최근에는 Telnet의 보얀취약성때문에 제조사에서 Telnet의 포트인 23번 포트를 막아둔다. 

- Telnet은 로그인할 때 비밀번호를 암호화하지 않고 그대로 전송하기 때문에 중간에서 해킹할 수 있다.
  - 보안 문제때문에 암호화를 하는 SSH(Secure Shell)가 만들어졌다.

# SSH
- SSH는 암호화를 해 데이터를 전송하기 때문에 중간에서 정보를 탈취하는게 어렵다.
- Key를 이용해 로그인할 수도 있다.
  - 이 방법을 사용하기 위해 두 개의 Key를 생성한다.
    - 하나는 외부에 절대 공개되선 안되는 Private Key, 개인 PC에만 저장돼야 한다.
    - 다른 하나는 외부에 공개되어도 되는 Public Key
      - 데이터를 암호화할 수 있다.
      - Private Key는 암호화된 데이터를 복호화할 수 있다.

## SSH 흐름
1. 우리는 멀리 있는 컴퓨터 B에 Public Key를 알려준다. 
2. 이후 우리의 컴퓨터 A에서 컴퓨터 B에 Key를 이용해 로그인을 시도한다면 
3. 컴퓨터 B는 랜덤한 문자열을 Public Key를 사용해 암호화하고 컴퓨터 A에 전송한다.
4. 컴퓨터 A가 올바른 Private Key를 가지고 있다면 이 암호화된 메세지를 복호화할 수 있다.
5. 컴퓨터 A는 복호화한 메세지를 컴퓨터 B에 전송하고
6. 컴퓨터 B는 복호화된 문자열이 암호화 전 문자열과 같다면 인증을 완료시킨다.
