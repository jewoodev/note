인터넷 기술을 알고나면, 다른 호스트와 네트워크 통신을 해서 데이터를 주고 받으려면 IP 주소를 알아야 한다는 것을 알게 된다. 그런데 우리가 웹브라우저를 사용하면서 '특정 사이트의 주소'라고 칭하는 것은 IP 주소가 아니었다. 대체 어떻게 사이트의 이름(문자열)으로 구성된 주소만으로 통신이 가능한걸까?

# 호스트명
사람이 IP 주소를 외우기란 쉽지가 않기 때문에 컴퓨터 과학자들은 **호스트명**이라고 컴퓨터의 IP마다 **이름**을 부여해 호스트를 구분했다. 그리고 그 호스트명을 IP 주소로 변환해주는 기술을 만들어 사람이 서핑하기 쉽도록 만들었다. 그럼 각 호스트가 다른 호스트의 호스트명과 IP 주소 값이 뭔지 알고 있어야 하는게 된다. 이를 위해서 인터넷의 전신인 ARPANET에서는 hosts 라는 이름의 파일을 만들어 데이터베이스에 저장해두고 ARPANET에 새로운 컴퓨터가 연결되면 파일을 업데이트했다. 그리고 모든 컴퓨터는 이 hosts 파일을 다운로드 받아서 업데이트했다. 

이 방법으로 사용자가 IP 주소를 외울 필요가 없어졌지만 ARPANET에서는 폭발적으로 늘어나는 사용자의 변화에 맞춰서 hosts 파일을 업데이트하고 배포하는 것이 힘들어졌다. 그래서 DNS라는 것이 만들어졌다. DNS는 사용자가 호스트명을 입력하면 자동으로 해당 호스트의 IP가 저장되어 있는 데이터베이스 서버로 연결되어 해당 호스트의 IP 주소를 알 수 있게 해주는 서비스이다. 즉, DNS를 사용하면 호스트가 다른 호스트명을 모두 가지고 있는게 아니라 분산된 데이터베이스에 필요할 때마다 물어본다. DNS 덕분에 네트워크에 많은 호스트가 연결되더라도 다른 기관에 알릴 필요가 없어져서 관리가 무척 편해졌다. 

여기서 호스트명이 아니라 **도메인명**이라는 개념이 등장한다.

# 도메인명
호스트명은 하나의 호스트 IP를 나타내는 이름이라면, 도메인명은 호스트들의 그룹을 나타내는 이름이라고 볼 수 있다.

> 호스트명은 `www.naver.com`에서 www에 해당하는 것으로 맨 앞에 위치하였으며 하나의 호스트를 나타냄. 즉, `mail.naver.com`도 호스트 하나 www도 호스트 하나를 나타내는 것이다. 그리고 naver.com이라는 도메인 명으로 묶인다.

도메인명은 어떤 구조로 되어 있을까? 

- 도메인명은 계층 구조로 설계되어 있음
- 도메인명은 점(.)으로 구분함
  - 제우.com, 제우.net ...
    - 점으로 구분한 것은 조직에 따라 구분한 것
    - e.g) com, net, gov, mil, org
      - 이런 것들을 최상위 도메인이라고 함
  - co.kr 을 보면 kr이 최상위 도메인이 됨
    - co가 2차 도메인이 됨
      - 서비스의 성격에 맞는 것으로 선택됨
  - 국제인터넷주소관리기구가 한국의 인터넷진흥원에 kr 최상위 도메인을 부여하고
    - 2차 도메인은 국제인터넷주소관리기구에서 신경쓰지 않아도 되고, 인터넷 진흥원이 관리할 수 있음

# 네임 서버
DNS는 도메인명으로 매핑한 IP 주소를 데이터베이스 서버에서 가져오는데, 이 서버를 **네임 서버**라고 한다.

웹브라우저에 www.naver.com를 입력하여 접속을 시도하면 아래와 같은 네임 서버 쿼리가 진행된다.

1. hosts 파일을 확인 
2. 가까운 네임 서버에 www.naver.com의 IP 주소를 물어봄
3. 가까운 네임 서버는 그걸 모르면 루트 네임 서버에 물어봄
4. 루트 네임 서버는 .com 네임 서버 주소를 알려줌
5. 가까운 네임 서버는 .com 네임 서버에 www.naver.com의 IP 주소를 물어봄
6. .com 네임 서버는 네이버 네임 서버의 주소를 알려줌
7. 가까운 네임 서버는 네이버 네임 서버에 www.naver.com의 IP 주소를 물어봄
8. 네이버 네임 서버는 www.naver.com의 IP 주소를 알려줌
9. 가까운 네임 서버가 내 컴퓨터에게 www.naver.com의 IP 주소를 알려줌
