Merge와 Rebase는 둘 다 A 브랜치의 작업을 B 브랜치로 합칠 때 사용하는 Git 명령어이다. 

둘의 가장 큰 차이는 '히스토리를 어떻게 남기느냐'에 있다.   
**Merge**는 '사실 그대로' 히스토리를 남기는 방식이다. A와 B 두 브랜치에서 각자 작업한 내용을 그대로 유지한 채, '두 브랜치를 합쳤다'는 새로운 기록(Merge Commit)을 하나 쿵 찍는다. 그래서 히스토리가 갈라졌다가 다시 합쳐지는 모양이 된다.  
**Rebase**는 '히스토리를 재정렬'하는 방식이다. A 브랜치의 작업 내역을 B 브랜치의 최신 작업 '위에' 차곡차곡 옮겨 붙여서, 마치 처음부터 B 브랜치 위에서 작업한 것처럼 히스토리를 한 줄로 깔끔하게 만든다. 그래서 히스토리가 매우 깨끗해진다는 장점이 있다.

둘의 차이를 여러 사람이 책을 쓰는 상황에서 비유할 수 있어요. Merge는 각자 쓴 챕터를 그대로 가져와서 마지막에 '홍길동과 이순신이 쓴 챕터를 합침'이라고 표기하는 방식이다. Rebase는 이순신이 쓴 챕터 위에 홍길동이 쓴 챕터를 순서대로 다시 타이핑해서, 마치 한 사람이 순서대로 쓴 것처럼 깔끔하게 정리하는 방식이다.
