## 1부. `Arrays.asList()` 핵심 정리

### 1) 반환 타입은 `java.util.Arrays.ArrayList` (특수 구현체)

이름은 같지만 우리가 흔히 쓰는 `java.util.ArrayList`가 **아니다**. `Arrays`의 내부 정적 중첩 클래스로, **고정 크기(fixed-size) 리스트**다.

```java
List<String> list = Arrays.asList("a", "b", "c");
list.set(0, "z");    // OK - 원소 교체는 가능
list.add("d");       // UnsupportedOperationException
list.remove(0);      // UnsupportedOperationException
```

### 2) 원본 배열과 메모리 공유 (백킹 배열)

배열을 복사하지 않고 **그대로 감싸기만** 한다. 한쪽을 바꾸면 다른 쪽도 바뀐다.

```java
String[] arr = {"a", "b", "c"};
List<String> list = Arrays.asList(arr);
arr[0] = "z";
System.out.println(list.get(0)); // "z"
```

### 3) 가변 컬렉션이 필요하면 한 번 더 감싸기

```java
List<String> mutable = new ArrayList<>(Arrays.asList(arr));
Queue<String> queue = new ArrayDeque<>(Arrays.asList(arr));  // 카드뭉치 문제처럼
Set<String> set = new HashSet<>(Arrays.asList(arr));
```

이렇게 컬렉션 생성자에 넘기면 내용을 복사한 **독립적인** 자료구조가 만들어진다.

### 4) 프리미티브 배열에 주의 ⚠️

`int[]`를 넘기면 `List<Integer>`가 아니라 **`List<int[]>` (원소 1개)**가 된다.

```java
int[] nums = {1, 2, 3};
List<Integer> wrong = Arrays.asList(nums);     // 컴파일 에러 또는 List<int[]>
Integer[] boxed = {1, 2, 3};
List<Integer> ok = Arrays.asList(boxed);       // OK
// 또는
List<Integer> ok2 = Arrays.stream(nums).boxed().toList();
```

### 5) `null` 허용 vs `List.of()` (Java 9+)

| 항목 | `Arrays.asList()` | `List.of()` |
|---|---|---|
| 크기 변경 | 불가 | 불가 |
| 원소 교체 (`set`) | **가능** | **불가** (완전 불변) |
| `null` 원소 | **허용** | **불허** (NPE) |
| 배열과 메모리 공유 | O | X (복사) |

> 진짜 불변 컬렉션이 필요하면 `List.of(...)`, 코딩테스트에서 빠르게 컬렉션 만들고 싶으면 `Arrays.asList(...)`을 다른 컬렉션 생성자에 넘기는 패턴이 안전하다.

### 6) 코딩테스트 관용구

```java
new ArrayDeque<>(Arrays.asList(arr))   // 큐/스택 초기화
new HashSet<>(Arrays.asList(arr))      // 빠른 lookup용 Set
Collections.sort(Arrays.asList(arr))   // 배열을 in-place 정렬 (원본 변경됨)
```

마지막 줄은 **백킹 배열 공유** 특성을 역으로 활용한 트릭이다 — 원본 배열까지 정렬되니 의도한 동작인지 확인하고 쓸 것.

## 2부. 컬렉션 초기화 시 사용할 수 있는 옵션들

### 1) `List.of()` / `Set.of()` / `Map.of()` (Java 9+) — **불변 컬렉션의 베스트**

진짜 읽기 전용 데이터에는 최우선 선택지.

```java
List<String> list = List.of("a", "b", "c");
Set<Integer> set = Set.of(1, 2, 3);
Map<String, Integer> map = Map.of("a", 1, "b", 2);
Map<String, Integer> bigMap = Map.ofEntries(
    Map.entry("a", 1),
    Map.entry("b", 2)
);
```

- **장점**: 진정한 불변, 메모리 효율적, 의도가 명확
- **주의**: `null` 불허, 추가/수정/삭제 모두 불가
- **언제**: 상수 데이터, 테스트 fixture, 수정이 절대 일어나지 않는 컬렉션

---

### 2) `new ArrayList<>(List.of(...))` 또는 `new ArrayList<>(Arrays.asList(...))` — **가변 컬렉션이 필요할 때**

```java
List<String> mutable = new ArrayList<>(List.of("a", "b", "c"));
Queue<String> queue = new ArrayDeque<>(List.of("a", "b", "c"));
Set<String> set = new HashSet<>(List.of("a", "b", "c"));
```

- **베스트**: `List.of()`로 감싸는 쪽이 `Arrays.asList()`보다 의도가 더 명확하고 안전하다 (백킹 배열 공유 이슈 없음).
- 단, `null`이 포함되어야 하면 `Arrays.asList()`만 가능.

---

### 3) `Stream.of(...).collect(...)` — **변환·필터·정렬이 함께 필요할 때**

```java
List<Integer> evens = Stream.of(1, 2, 3, 4, 5)
    .filter(n -> n % 2 == 0)
    .collect(Collectors.toList());      // 가변 (구현체 비보장)

List<Integer> sorted = Stream.of(3, 1, 2)
    .sorted()
    .toList();                          // Java 16+, 불변
```

- **언제**: 단순 초기화가 아니라 가공이 함께 필요할 때만.
- **주의**: 단순 초기화 용도로 쓰면 오버엔지니어링.

---

### 4) Double-brace 초기화 — **❌ 안티 패턴 (쓰지 말 것)**

```java
// 절대 쓰지 말 것
List<String> list = new ArrayList<>() {{
    add("a"); add("b"); add("c");
}};
```

- 매번 익명 내부 클래스를 만들어 외부 클래스 참조를 잡고 있어 **메모리 누수** 위험.
- 직렬화 문제, 디버깅 어려움.
- 과거 일부 코드에서 보일 수 있지만 **현대 자바에서는 금기**.

---

### 5) `Collections.unmodifiableList(...)` 등 — **레거시 호환용**

```java
List<String> readOnly = Collections.unmodifiableList(
    new ArrayList<>(List.of("a", "b"))
);
```

- **언제**: Java 8 이하 환경, 또는 기존 가변 컬렉션을 읽기 전용 뷰로 노출할 때.
- Java 9+에서는 새로 만드는 거라면 `List.of()`가 우선.

---

### 6) `EnumSet` / `EnumMap` — **enum 키에 특화**

```java
EnumSet<DayOfWeek> weekend = EnumSet.of(DayOfWeek.SATURDAY, DayOfWeek.SUNDAY);
EnumMap<DayOfWeek, String> map = new EnumMap<>(DayOfWeek.class);
```

- enum 전용으로 비트벡터 기반이라 **`HashSet`보다 훨씬 빠르고 메모리 효율적**.
- enum 키라면 무조건 이걸 쓰는 게 베스트.

---

## 3부. 결정 트리

```
불변이면 충분한가?  ─ Yes ─→ List.of() / Set.of() / Map.of()
       │
       No (수정 필요)
       │
       ├─ enum 키?            ─→ EnumSet / EnumMap
       ├─ Queue/Deque/Set?    ─→ new ArrayDeque<>(List.of(...)) 같은 형태
       ├─ 변환·필터 필요?       ─→ Stream.of(...).collect(...)
       └─ 일반 List/Set/Map    ─→ new ArrayList<>(List.of(...))
```

## 4부. 코딩테스트 관용구 정리

```java
// 불변 데이터 (lookup table 등)
Map<Character, Integer> priority = Map.of('+', 1, '-', 1, '*', 2, '/', 2);

// 빠른 멤버십 검사용 Set
Set<String> vowels = Set.of("a", "e", "i", "o", "u");

// BFS/DFS용 Queue/Deque 초기화
Deque<int[]> queue = new ArrayDeque<>(List.of(new int[]{0, 0}));

// 정렬·가공이 필요한 가변 리스트
List<Integer> sorted = new ArrayList<>(List.of(3, 1, 2));
Collections.sort(sorted);
```

**핵심 원칙**: *"수정이 필요한가?"*를 먼저 묻고, 필요 없으면 `List.of()` 계열을 쓰는 것이 현대 자바의 베스트 프랙티스다.
