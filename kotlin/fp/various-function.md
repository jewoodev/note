# 확장 함수
- 확장 함수 vs 멤버 함수
  - => 멤버 함수가 우선적으로 호출된다.
- 확장 함수가 오버라이딩되면 그 중 어떤게 호출될까?
  - 해당 변수의 **현재 타입**을 기준으로 호출된다.
  - => 정적인 타입에 의해 어떤 확장 함수가 호출될지 결정된다.

확장 함수의 개념은 확장 프로퍼티로 연결된다. 그래서 자바에서 작성된 getter를 코틀린에서 custom getter를 만드는 것처럼 프로퍼티로 확장할 수도 있다.

# infix 함수 (중위 함수)
함수를 호출하는 새로운 방법이다.

```kotlin
fun Int.add(other: Int): Int = this + other

infix fun Int.add2(other: Int): Int = this + other

3.add(4)

3.add2(4)
3 add2 4
```

# inline 함수
함수가 호출되는 대신 함수를 호출한 지점에 함수 본문을 그대로 복붙하는 기능이다.

이 기능은 왜 사용할까? 함수를 파라미터로 전달할 때 오버헤드를 줄일 수 있다. 하지만 이는 성능 측정을 거쳐 신중하게 사용되어야 한다.
