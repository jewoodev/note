자바에서는 내부 클래스가 static이 될 수도, 되지 않을 수도 있다. 이펙티브 자바 3판 아이템 24, 86에서는 다음과 같은 이야기를 우리에게 건넨다.

1. 내부 클래스는 숨겨진 외부 클래스 정보를 가지고 있어, 참조를 해지하지 못하는 경우 메모리 누수가 발생할 수 있고, 이를 디버깅하기 어렵다.
2. 내부 클래스의 직렬화 형태가 명확하게 정의되지 않아 직렬화에 한계가 있다.

즉, 내부 클래스가 외부 클래스를 참조함으로 인해 생기는 문제점들이 있다. 그래서 이펙티브 자바에서 내부 클래스를 만들 땐 static을 쓰지 않는 내부 클래스 대신에 static을 사용하는 내부 클래스를 사용하라고 가이드를 말해준다. 코틀린은 이 가이드를 충실히 따르고 있다.
