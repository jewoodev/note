# 비트마스크
'하나의 정수'로 집합을 나타내는 것을 말한다. 이게 무슨 말인지 이해할 수 있도록 찬찬히 이야기를 이어나가겠다.

집합은 중복되는 수가 없다. 그리고 집합을 나타내기 위해 '하나 이상의 정수'가 필요하다. 집합은 중복되지 않기 때문에 없으면 0, 있으면 1로 표현이 가능하다. 이 특성을 이용해 2진수로 집합을 표현해보자.

`{1, 3, 4, 5, 9}` 집합을 2진수로 표현하면, $2^{1}\ +\ 2^{3}\ +\ 2^{4}\ +\ 2^{5}\ +\ 2^{9}$가 된다. 즉, 570이 된다. 

이게 비트마스크의 개념이다. `{1, 3, 4, 5, 9}`를 비트마스크를 적용해 570으로 표현할 수 있는 것이다. 

## 장점
- 공간을 적게 사용함 (항상 정수 1개를 이용해 표현 가능하기에)
- 정수라는 점, 리스트에 집합을 저장할 때 특정한 자료형을 사용하는 등의 복잡한 과정이 필요하지 않게 됨
- 집합에 필요한 연산의 성능이 뛰어남, 아래의 세 가지 모두 O(1)
  - 검사
  - 추가
  - 삭제

## 조건
- 집합에 저장할 수 있는 수의 범위가 정해져 있어야 함
  - 2진수를 사용하기에 100이 들어오면 저장할 수 없는 크기의 수가 되버리기 때문
  - 32비트, 64비트의 정수형이 각 언어에서 사용됨
    - 각각 0 ~ 31, 0 ~ 63 까지 저장 가능

## 활용가이드
- n이 포함되어 있는지 검사
  - S & (1 << n) = 2의 n 승이 나오면 포함 o, 아니면 x / 실제로는 0인지 0이 아닌지로 연산함
- n 추가하기
  - S | (1 << n)
- n 제거하기
  - S & ~(1 << n)
  - S ^ (1 << n) -> 이 경우엔 이미 없는데 연산하면 추가됨
- 1부터 n 까지의 전체 집합
  - (1 << (n + 1)) - 1
- 공집합
  - 0
