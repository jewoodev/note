# 변경 관점에서의 응집도
- 변경 관점에서 응집도는 모든 내부 요소들이 함께 변경되는 정도를 의미한다.
  - 모듈 안의 모든 요소들이 **동일한 이유**로 함께 변경된다면 "응집도가 높다."
  - 모듈 안의 모든 요소들이 **서로 다른 이유**로 변경된다면 "응집도가 낮다."

## 속도와 시점
변경 관점에서 응집도를 이야기할 땐 변경의 속도와 시점이 중요하다. 모듈의 전체 코드가 동일한 시점에 동일한 속도로 변경되면 응집도가 높은 것이다. 반대로 서로 다른 시점에 서로 다른 속도로 변경되면 응집도가 낮은 것이다. 

객체 지향적인 방식으로 작성된 코드는 응집도가 높아 단 하나의 이유만으로 변경된다. 이는 SOLID의 제 1원칙 단일 책임 원칙과 관련이 있다. 이 원칙은 응집도가 높은 클래스를 만들기 위해 서로 다른 이유로 변경되는 코드를 서로 다른 클래스로 분리해야 한다는 사실을 강조한다. 동일한 시점에 동일한 속도로 변경되게 하려면 단 하나의 이유만으로 변경되게 만들어야 한다. 객체의 크기를 정할 때도 변경의 이유를 기준으로 크기를 정하자.
