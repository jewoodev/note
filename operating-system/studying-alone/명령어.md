# 명령어의 구조
명령어는 일반적으로, 크게 표현하자면 연산코드와 오퍼랜드로 이루어져 있다. 

- 연산코드: 수행할 연산을 담고 있는 코드
  - 연산코드가 담고 있는 내용은 CPU마다 다름
- 오퍼랜드: 연산에 사용될 데이터 혹은 데이터가 저장된 위치
  - 전자와 후자 중에 후자가 오퍼랜드에 훨씬 더 자주 저장됨. 그래서 오퍼랜드가 담기는 그 공간, 즉 오퍼랜드 필드를 주소 필드라고도 부름.
    - 데이터보다 데이터가 저장된 위치가 더 자주 저장되는 이유는, 명령어 내에서 표현할 수 있는 데이터 크기가 제한적이기 때문이다.
    - 관련된 용어로 '유효주소(effective address)'가 있다.
      - 이 용어는 '연산에 사용될 데이터가 저장된 위치'를 의미한다.
  - 오퍼랜드는 하나의 명령어에 하나도 없을 수도, 여러 개가 있을 수도 있다.
  - CPU 입장에서 오퍼랜드에 연산에 사용될 데이터가 주소가 있든, 연산에 사용될 데이터를 찾아낼 수 있어야 한다. 그 방법이 '명령어 주소 지정 방식'이다.

## 명령어 주소 지정 방식(addressing mode)
- 연산에 사용될 데이터가 저장된 위치를 찾는 방법
  - = 유효 주소를 찾는 방법
- 다양한 주소 지정 방식들이 있다
- 이 또한 CPU 종류마다 차이점이 있다

### 즉시 주소 지정 방식(immediate addressing mode)
- 연산에 사용할 데이터를 오퍼랜드 필드에 직접 명시
- 가장 간단한 형태의 주소 지정 방식
- 이 방식으로 지정 가능한 '연산에 사용할 데이터의 크기'가 작아지지만, 빠름

## 직접 주소 지정 방식(direct addressing mode)
- 오퍼랜드 필드에 유효 주소를 직접적으로 명시
- 유효 주소를 표현할 수 있는 데이터의 크기가 연산 코드만큼 줄어듬

## 간접 주소 지정 방식(indirect addressing mode)
- 오퍼랜드 필드에 유효 주소의 주소를 명시
- 앞선 방식들에 비해 속도가 느림
  - CPU가 메모리를 읽어들이는 속도는 CPU의 연산 속도에 비해 굉장히 느리기 때문에 메모리 접근을 최소화하는 것이 속도면에선 무조건 좋음

## 레지스터 주소 지정 방식(register addressing mode)
- 연산에 사용될 데이터가 저장된 레지스터를 명시
- 메모리에 접근하는 것보다 레지스터에 접근하는 것이 속도가 훨씬 빠름
  - 직접 주소 지정 방식에 비해 조금 더 빠를 수 있음

## 레지스터 간접 주소 지정 방식(register indirect addressing mode)
- 연산에 사용될 데이터를 메모리에 저장하고, 그 유효주소를 레지스터에 저장하고, 그 레지스터를 명시
