PCB들은 연결리스트 구조로 저장된다. 운영체제는 프로세스가 종료되면 연결리스트에서 해당 프로세스의 PCB를 제거한다. 

# PCB에 담기는 정보
- 포인터
  - 부모와 자식 프로세스에 대한 포인터
  - 할당된 자원에 대한 포인터
  - 프로세스의 상태가 다른 상태로 전환될 때 저장하는 포인터
- 프로세스 상태
  - 생성, 준비, 실행, 대기, 완료 / 5가지 상태를 가짐
- 프로세스 ID
  - 프로세스를 식별하기 위한 숫자
- 프로그램 카운터
  - 다음으로 실행할 명령어의 주소를 포함하는 정보
  - 컨텍스트 스위칭이 일어날 때 원래 실행하던 흐름을 그대로 이어서 실행하기 위해 꼭 필요
- 레지스터 정보
  - 프로세스가 실행될 때 사용했던 레지스터 값들이 저장됨
- 메모리 관련 정보
  - 프로세스의 메모리 위치 정보, 메모리 침범을 막기 위한 경계 레지스터 값 등등
- CPU 스케줄링 정보
  - 스케줄링에 필요한 우선순위, 최종 실행시간, CPU 점유시간 등등
