# 1940년도
최초의 컴퓨터 애니악이 미군지휘 하에 펜실베니아 대학교에서 개발되었다. 최초의 목적은 미사일 탄도 계산을 위한 것, 세계에서 가장 큰 스케일의 전자 디지털 계산기였다. 애니악은 특정 명령에 맞는 스위치와 배선을 연결해서 프로그래밍했다. 큰 기기에 많은 스위치와 배선을 이용하기 때문에 많은 인력이 필요했고 시간이 많이 들었다. 입출력을 위해 IBM 펀치 카드를 이용했는데 입출력 속도가 굉장히 느리고 입출력 도중에는 계산을 할 수도 없었다. 애니악의 무게는 30톤이었다. 애니악은 하드웨어 비용이 굉장히 비싸게 쓰여 CPU를 어떻게 최대한 많이 사용할 수 있을지 고민하게 되었다.

# 1950년도 초
진공관과 전선으로 만든 논리회로로 아주 작은 크기로 만든 직접회로가 개발됐다. 이로써 현대적인 컴퓨터의 모습을 갖추게 되었다. 이땐 CPU와 메모리가 있었지만 키보드와 모니터는 없었다. 

# 1950년도 중후반
이전까지만 해도 프로그래머가 펀치카드를 이용해 프로그램을 작성하면 오퍼레이터가 직접 컴퓨터에 카드를 넣고 결과가 나오면 다시 프로그래머에게 전달해주었다. 이렇게 하니까 컴퓨터의 처리속도보다 오퍼레이터가 카드를 넣고 전달하는 과정이 상대적으로 더 느렸다. 그래서 펀치카드 여러 개를 한 번에 컴퓨터에 전달해주면 컴퓨터는 여러 개의 프로그램을 순서대로 실행해서 결과도 한 번에 확인할 수 있도록 개발했는데 이를 싱글 스트림 배치 시스템이라고 부른다. 이를 통해 CPU 사용률이 올라갔다.

여기서 컴퓨터 시스템 개발자들은 CPU의 사용률을 더 올리기 위해 고민했다. 당시엔 입출력 작업을 하는 동안엔 CPU를 사용할 수 없었는데, 입출력을 담당하는 I/O Device Controller를 만들어 입출력 중에도 CPU가 계산할 수 있도록 만들었다. I/O Device Controller는 입출력 작업이 끝나면 CPU에게 Interrupt 신호를 주고 CPU는 그 신호를 받으면 다시 처리하는 식으로 동작했다.

# 1960년도
싱글 스트림 배치 시스템의 한계를 극복했다. 프로그램을 순서대로 하나씩 실행하는 것이 아니라 메모리에 여러 프로그램을 올려놓고 시간을 나눠서 A프로그램 0.1초, B프로그램 0.1초, C프로그램 0.1초, 이렇게 빠르게 돌아가면서 실행시킨다. 매우 빠르게 번갈아가면서 실행하기 때문에 사용자에게는 전부 동시에 실행되는 것처럼 보이게 된다. 이런 시스템을 분할 시스템(time-sharing system) 이라고 부른다. 이를 통해 CPU 사용량이 더 올라가게 되었다.

프로그램을 동시에 여러 개 실행시킬 수 있게 되어 여러 사용자가 한 번에 사용 가능하게 되었다. 컴퓨터는 비쌌기 때문에 하나의 컴퓨터를 두고 터미널이라는 단말기를 통해 여러 사용자들이 여러 개의 터미널로 하나의 컴퓨터에 접근해서 사용했다. 사용자는 컴퓨터를 혼자 쓰는 것 같아서 컴퓨터에 개인 문서 같은 개인정보를 저장하기 시작했고, 이 때문에 파일 시스템이 등장했다.

또 다른 의미에서 1960년대는 혼란의 시기였다. 여러 프로그램을 동시에 실행시키니 생각지도 못한 문제들이 발생했기 때문이다. 

1. 첫 번째는 여러 프로그램이 올라와서 작업을 실행함으로써 메모리 침범 이슈다. 
2. 또 다른 문제는 기존에 프로그램을 작성할 때 메모리에는 프로세스가 하나만 있기 때문에 
   1. 메모리 주소를 갖고 올라가 있는 프로세스가 하나라고 가정하고 프로그래밍을 했는데 
   2. 이제는 여러 개가 올라오니까 자기가 어느 메모리 위치에서 실행되는지 모르게 되는 이슈다.
   3. 그래서 하드웨어 적으로 base register라는 것을 추가해서 프로그램의 시작 주소를 저장하고 
      4. 모든 프로그램은 0번지에서 실행한다고 가정하게 된다.
3. CPU 사용률을 높이고 효율성을 높이면서 발생하는 문제들을 해결하다보니
   1. 미국의 대학들은 이것이 굉장히 중요하고 복잡하다는 것을 깨닫고
   2. 하나의 학문으로 자리잡게 했다.
