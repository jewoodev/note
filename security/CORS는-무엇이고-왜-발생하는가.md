CORS는 웹 브라우저가 보안을 위해 다른 출처(도메인)의 리소스 접근을 제한하는 정책이에요. 예를 들어 localhost:3000에서 실행 중인 프론트엔드가 api.example.com의 API를 호출하면, 브라우저는 "다른 출처니까 위험할 수 있어!"라며 요청을 막아버립니다.

이런 제한이 없다면 악의적인 사이트가 여러분의 인증 정보를 몰래 가로채서 다른 사이트의 API를 마음대로 호출할 수 있겠죠. CORS는 이를 방지하기 위한 브라우저의 보안 메커니즘입니다.

'아파트 경비실'에 비유하면 쉬워요. 우리 아파트(같은 출처) 주민은 자유롭게 드나들 수 있지만, 다른 아파트(다른 출처) 사람이 방문하려면 경비(브라우저)가 "이 방문자를 허용하시나요?"라고 주민(서버)에게 확인해야 하는 거죠. 서버가 "Access-Control-Allow-Origin" 헤더로 "허용!"이라고 답해야 통과할 수 있습니다.
