- CORS는 브라우저의 Same-Origin Policy에 의해 발생하는 교차 출처 요청 제한을, 서버가 명시적으로 허용해주는 메커니즘
- 브라우저는 보안을 위해 현재 페이지의 Origin — 즉 프로토콜, 호스트, 포트 조합 — 과 다른 출처로의 요청을 기본적으로 차단
- 이를 Same-Origin Policy라고 하는데, 악의적인 사이트가 사용자의 인증 정보를 이용해 다른 서비스에 요청을 보내는 것을 막기 위한 정책
- 현대 웹에서는 프론트엔드와 백엔드의 출처가 다른 경우가 일반적이기 때문에, 서버가 응답 헤더를 통해 '이 출처의 요청은 허용한다'고 선언하는 방식이 필요
  - 그 선언 방식이 CORS
- 구체적으로는 `Content-Type: application/json` 같은 헤더가 포함된 요청의 경우, 브라우저가 먼저 OPTIONS 메서드로 Preflight 요청을 보내서 서버의 허용 여부를 확인하고, 허용되면 실제 요청을 보내는 2단계로 동작
- 중요한 점은 CORS는 브라우저 레벨의 정책이라서, 서버 간 통신에서는 적용되지 않음
