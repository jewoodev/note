# 카프카 브로커
- 프로듀서로 데이터를 보내고, 컨슈머가 데이터를 가져갈 때 반드시 통신하게 되는 프로세스
- 데이터를 분산 저장해서 장애가 발생하더라도 안정성을 유지하게 해준다.
- 일반적으로 하나의 서버에 하나의 브로커 프로세스가 실행됨
- 브로커 1개로도 기본 기능은 사용할 수 있지만 데이터를 안전하게 보관하고 처리하기 위해 3대 이상의 브로커 서버를 클러스터로 묶어서 운영한다.
- 클러스터로 묶인 브로커들은 프로듀서가 보낸 데이터를 안전하게 분산 저장하고 복제하는 역할 수행

# 주키퍼
- 카프카 클러스터를 운영하기 위해 반드시 필요한 애플리케이션
- 2점 버전대까지는 반드시였지만 3점대부터는 상관없다고 알려짐
  - 3점대에서도 주키퍼를 완벽히 대체하고 있지 못하는 부분이 있어, 사용하는 것이 권장됨
  - 대부분의 환경에서 사용중

# 카프카 클러스터
- 하나의 클러스터는 하나 이상의 브로커로 이루어짐
  - 한 개의 브로커는 하나의 physical machine이나 서버, 혹은 인스턴스에서 동작함
- 상용 환경에서는 하나의 클러스터가 최소 3개의 브로커를 갖도록 운영
- 데이터 양이 많고 확장성이 중요한 경우엔 50~100개까지로도 운영
- 용도에 따라 클러스터를 2개, 3개까지 나눠서 운영하기도 함
